# **Project Multiverse — AI Prompt Patterns**

These are the **canonical prompt structures** used by structural AIs working within Project Multiverse.  
They ensure all AI‑assisted work remains:

- continuity‑first  
- RTT‑aligned  
- drift‑bounded  
- reproducible  
- world‑safe  
- student‑friendly  

All prompts must follow the Multiverse session context.

---

# **1. W‑Operator Prompts (RTT/1)**

### **1.1 Define a W‑Operator**
```
Define the world operator <OPERATOR_NAME>.

Include:
- operator family (Spawn, Frame, Evolve, Structure, Bridge, Transfer, Verify)
- purpose
- constraints
- failure modes
- continuity boundaries
- RTT/1 grammar alignment
- RTT/2 stacking behavior
- RTT/3 drift detection rules
```

### **1.2 Analyze Operator Behavior**
```
Analyze the behavior of <OPERATOR_NAME> under the following conditions:
<CONDITIONS>

Include:
- expected behavior
- drift risks
- resonance interactions
- continuity implications
- safety boundaries (X-operators)
```

### **1.3 Compare Two W‑Operators**
```
Compare <OPERATOR_A> and <OPERATOR_B>.

Include:
- functional differences
- continuity-preservation differences
- stacking differences
- resonance implications
- safety considerations
```
